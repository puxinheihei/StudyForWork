# Java 练手小项目需求：个人任务管理系统

> 目标：通过从 0 到 1 完成一个简单但完整的 Java Web 项目，训练后端编码能力和工程习惯，为暑期实习面试提供可讲的“小而真实”的项目支撑。

---

## 一、项目概述

- 项目名称：个人任务管理系统
- 核心功能：为单个用户管理日常待办任务，支持任务的创建、查看、更新、删除与简单统计查询。
- 技术目标：
  - 使用 Java + Spring Boot 实现后端接口。
  - 使用 MySQL 持久化数据。
  - 具备基础的异常处理、日志记录、参数校验和简单安全控制。
  - 编写少量单元测试，提升代码可维护性。

---

## 二、技术栈与约束

- 后端：
  - 语言：Java
  - 框架：Spring Boot（包含 Spring Web、Spring Data JPA 或 MyBatis 均可）
- 数据库：
  - MySQL（建议本地安装一个轻量实例）
- 其他：
  - 构建工具：Maven 或 Gradle（二选一）
  - 日志：SLF4J + Logback（Spring Boot 默认即可）
  - 接口测试：Postman 或 curl
  - 单元测试：JUnit（Spring Boot 默认集成）

约束要求：

- 所有核心代码由自己编写，可以使用 AI 辅助查资料或给出建议，但最终实现要自己理解并能够讲清楚。
- 接口风格使用 RESTful，数据交互使用 JSON。
- 提供一键启动方式（通过 `main` 方法或 `spring-boot:run`）。

---

## 三、功能需求

### 1. 任务基础功能

- 创建任务：
  - 字段示例：
    - 标题（必填）
    - 描述（选填）
    - 截止日期（选填）
    - 优先级（可选值：低/中/高）
    - 状态（未开始 / 进行中 / 已完成）
  - 行为：
    - 创建时自动生成任务 ID 和创建时间。

- 查看任务列表：
  - 支持分页（例如：page、pageSize）。
  - 支持按状态筛选（可选）。
  - 按创建时间或截止日期排序（可选）。

- 查看单个任务详情：
  - 通过任务 ID 获取完整信息。

- 更新任务：
  - 支持修改标题、描述、截止日期、优先级、状态。
  - 更新时记录更新时间。

- 删除任务：
  - 通过任务 ID 删除任务。

### 2. 统计与查询功能（可选增强）

- 查询当天/本周未完成任务数量。
- 查询已完成任务列表。
- 根据关键字搜索任务标题或描述。

---

## 四、接口设计示例

可以按 REST 风格设计接口路径，示例：

- 创建任务：`POST /api/tasks`
- 获取任务列表：`GET /api/tasks`
  - 支持查询参数：
    - `status`（可选）：按状态过滤
    - `page`、`pageSize`（可选）：分页
- 获取任务详情：`GET /api/tasks/{id}`
- 更新任务：`PUT /api/tasks/{id}`
- 删除任务：`DELETE /api/tasks/{id}`

要求：

- 请求和响应使用 JSON。
- 对非法参数返回统一格式的错误响应（包含错误码和错误信息）。

---

## 五、数据库设计

建议至少设计以下表：

### 表：`task`

- 字段示例：
  - `id`：主键，自增
  - `title`：任务标题，非空
  - `description`：任务描述，可空
  - `status`：任务状态（枚举：TODO/DOING/DONE）
  - `priority`：优先级（枚举：LOW/MEDIUM/HIGH）
  - `due_date`：截止日期，可空
  - `created_at`：创建时间，非空
  - `updated_at`：更新时间，非空

后续如需支持多用户，可以新增 `user` 表并在 `task` 中增加 `user_id` 字段。

## 六、非功能性要求

### 1. 异常处理

- 为常见异常提供统一处理：
  - 参数校验失败（例如标题为空）；
  - 任务不存在（访问不存在的任务 ID）。
- 返回统一结构，如：
  - `code`：错误码（如 0 表示成功，非 0 表示各种错误）；
  - `message`：错误信息；
  - `data`：有错误时可为 null。

### 2. 日志

- 为关键操作打日志：
  - 创建任务、更新任务、删除任务。
- 日志应包含时间、操作类型、任务 ID 等信息。

### 3. 参数校验

- 对创建/更新任务的请求体进行基本校验：
  - 标题非空；
  - 优先级、状态只能是允许的取值之一；
  - 截止日期格式正确。

## 七、测试与验证

- 单元测试：
  - 至少为任务 Service 层编写 2–3 个单元测试用例。
  - 覆盖创建任务、更新状态、查询列表等核心逻辑。

- 手动验证：

    - 正常路径：功能按预期工作。
    - 错误路径：错误响应清晰、不会崩溃。

## 八、分阶段实现建议

建议按以下顺序实现，方便与你的每日计划对齐：

1. 初始化项目：
   - 搭建 Spring Boot 工程，跑通 `/health` 接口。
2. 实现任务实体与内存版 CRUD：
   - 不接数据库，先在内存中用 List/Map 管理任务。
3. 接入 MySQL：
   - 建表、配置数据源，将 CRUD 改为真实持久化。
4. 增加参数校验与统一异常处理：
   - 使用 Spring Validation 或手动校验。
5. 增加日志与简单统计接口：
   - 记录关键操作日志，实现 1–2 个统计查询。
6. 补充单元测试与整理文档：
   - 为核心逻辑编写测试；
   - 在本文件基础上补充一份“项目总结”，为面试准备素材。
